# 📚 Project 01 — Library Management System

> Learn **ASP.NET Core MVC** by following the complete HTTP Request Lifecycle, then build a real-world Library Management System from scratch.

---

## 🎯 Project Goal

This project is **not** just about building a CRUD application.

The goal is to understand **how ASP.NET Core MVC works internally**, then apply that knowledge while building a complete project.

Instead of memorizing code, we will understand:

- How the HTTP Request enters the application.
- What happens inside ASP.NET Core.
- Why each component exists.
- How every layer communicates with the next one.

By the end of this project, I should be able to explain every step of the Request Lifecycle and confidently build production-ready MVC applications.

# 📖 Learning Philosophy

This project follows one simple rule:

> **Understand first. Build second. Optimize later.**

Every Sprint focuses on **one concept only**.

The goal is not to finish the project quickly.

The goal is to understand **why every line of code exists** before writing it.

# 📚 Learning Phases

## 🚧 Phase 0 — The Complete Request Lifecycle
    Learn how an HTTP Request travels inside ASP.NET Core from the Browser until the HTML Response returns.

    Build the complete application while applying everything learned in Phase 0.

    Secure the application using Authentication, Authorization and ASP.NET Core Identity.
