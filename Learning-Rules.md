# Learning Rules

These are the rules that will guide this journey.

## Rule 1

Understand before coding.

---

## Rule 2

Never copy code without understanding it.

---

## Rule 3

Always ask:

* Why?
* How?
* When?
* Why not another solution?

---

## Rule 4

Every feature starts with design.

---

## Rule 5

Write clean code, not just working code.

---

## Rule 6

Refactor after implementation.

---

## Rule 7

Mistakes are part of learning.
Fix them, document them, and don't repeat them.

---

## Rule 8

Every Sprint ends with:

* Code Review
* Interview Questions
* Git Commit
* Roadmap Update
