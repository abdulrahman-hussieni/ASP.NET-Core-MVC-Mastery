# Learning Rules

> These are the rules that will guide this learning journey from the first project to the last.

---

# Rule 1 — Understand Before Coding

Never write code before understanding:

* Why this concept exists.
* What problem it solves.
* How it works internally.
* When to use it.
* When not to use it.

---

# Rule 2 — Learn the "Why", not only the "How"

Don't memorize syntax.

Always understand the reasoning behind every decision.

---

# Rule 3 — Think Before Opening Visual Studio

Every feature starts with planning.

Before writing code, answer:

* What am I building?
* Why am I building it?
* How should it be designed?

---

# Rule 4 — Build Like a Software Engineer

The objective is not to finish the project quickly.

The objective is to build software that is:

* Clean
* Readable
* Maintainable
* Scalable

---

# Rule 5 — One Concept at a Time

Never learn multiple new concepts in the same lesson.

Master one concept before moving to the next.

---

# Rule 6 — Every Lesson Ends with Practice

Each lesson should include:

* A practical implementation
* A small challenge
* Interview questions
* Code review

Learning without practice is incomplete.

---

# Rule 7 — Mistakes Are Part of the Journey

Making mistakes is expected.

Every mistake should help improve understanding.

Never skip debugging.

---

# Rule 8 — Refactor Before Moving Forward

Working code is not always good code.

Before finishing any Sprint:

* Improve naming
* Improve readability
* Remove duplication
* Apply best practices

---

# Rule 9 — Progress Over Perfection

The goal is continuous improvement.

Every project should be better than the previous one.

---

# Rule 10 — Enjoy the Journey

This repository is not just about finishing four projects.

It is about becoming a better software engineer.

Take your time.

Ask questions.

Stay curious.

Keep learning.
