# Learning Workflow

> Every project in this repository follows the same workflow.

The goal is not to build applications as fast as possible.

The goal is to understand how professional software is designed, implemented, reviewed, and improved.

---

# Step 1 — Learn

Before writing any code:

* Understand the concept.
* Learn why it exists.
* Learn how it works internally.
* Explore real-world use cases.

---

# Step 2 — Design

Before opening Visual Studio:

* Define the requirements.
* Design the database.
* Plan the architecture.
* Decide how the feature should work.

---

# Step 3 — Build

Implement only the current feature.

Focus on:

* Clean Code
* Readable Code
* Best Practices

Avoid unnecessary complexity.

---

# Step 4 — Review

After implementation:

* Review the code.
* Improve naming.
* Improve folder structure.
* Remove duplicated code.
* Discuss possible improvements.

---

# Step 5 — Refactor

Refactor the implementation when needed.

The first solution is not always the best solution.

Learning to improve code is part of becoming a software engineer.

---

# Step 6 — Interview Preparation

At the end of every lesson:

* Answer interview questions.
* Explain the concept in your own words.
* Solve a small challenge.

If you can explain it, you understand it.

---

# Step 7 — Commit

Commit the completed work with a meaningful commit message.

Example:

```bash
Add Book CRUD functionality

Implement Author entity

Configure Entity Framework Core
```

---

# Step 8 — Update Progress

After finishing the Sprint:

* Update ROADMAP.md
* Update the project README
* Mark completed concepts

This keeps the learning journey organized and easy to review later.

---

# Learning Cycle

```text
Learn
   ↓
Design
   ↓
Build
   ↓
Review
   ↓
Refactor
   ↓
Interview
   ↓
Commit
   ↓
Update Progress
   ↓
Repeat
```

---

# Remember

The objective is not to finish quickly.

The objective is to understand deeply, improve continuously, and become a better ASP.NET Core developer with every Sprint.
